# AP Crypto Club — Telegram Mini App + Админ-панель

Экосистема клуба AP Crypto: Telegram Mini App для резидентов, браузерная
админ-панель (CRM + геймификация + рассылки) и API-сервер с общей базой на
Supabase. Регистрация и уведомления идут через бота **@apclubnews_bot**.

> Этот README — актуальный срез проекта на **02.07.2026** и точка отсчёта для
> дальнейшей стратегии внедрения и усиления решения (см. раздел
> [«Точки роста»](#точки-роста--стратегия)).

## 🏗 Архитектура

Три независимых приложения + общая БД + бот:

```
                         ┌─────────────────────────┐
   Telegram ──/start──▶  │  bot (@apclubnews_bot)  │
                         │  приветствие, вход,     │
                         │  DM-уведомления         │
                         └────────────┬────────────┘
                                      │
  ┌────────────────┐          ┌───────▼────────┐          ┌────────────────┐
  │  Mini App      │  /api    │   API-сервер   │  jsonb   │    Supabase    │
  │  (frontend/)   │◀────────▶│   (server/)    │◀────────▶│  profiles      │
  │  React+Vite    │          │  Node+Express  │          │  (Postgres)    │
  └────────────────┘          └───────▲────────┘          └────────────────┘
                                      │  /api/admin
                              ┌───────┴────────┐
                              │  Админ-панель  │
                              │  (admin/)      │
                              │  React+Vite    │
                              └────────────────┘
```

| Папка | Стек | Роль |
|-------|------|------|
| `frontend/` | React 19 + Vite + TS | Mini App резидента (профиль, достижения, лидеры, витрина, бадди, календарь) |
| `admin/` | React + Vite + TS | CRM резидентов, геймификация, уведомления, дашборд (вход по паролю) |
| `server/` | Node + Express (`--experimental-strip-types`, Node ≥ 22.6) | API, валидация Telegram `initData`, бот, планировщик уведомлений |
| Supabase | Postgres | Единое хранилище профилей (`profiles`, jsonb-колонка `data`) |

**Абстракция хранилища** (`server/src/store.ts`) отделена от роутов: файловый
фолбэк или Supabase выбирается по наличию `SUPABASE_URL`/`SUPABASE_SERVICE_KEY`.

## ✅ Что уже реализовано

### Mini App (резидент)
- **Онбординг** (`public/onboarding.html`) — мобильная колонка + инфографика (метрики клуба, сравнение активный/неактивный, таймлайн, темы чатов).
- **Гейт регистрации** — при первом входе экран Register (имя/фамилия из Telegram + почта для проверки доступа) → `POST /api/profile/register`.
- **Профиль** — аватар/имя из Telegram, «о себе», прогресс разблокировки N/12, меню (Достижения, Таблица лидеров, Витрина клуба), «Продлить подписку».
- **Редактирование профиля** — все поля, дата-пикер 18+, загрузка фото (dataURL 256px), соцсети, тумблер «Разрешить писать».
- **Достижения и роли** — money-достижения (получено/нет) + роли с прогрессом по тирам.
- **Сообщество** — список резидентов + карточка участника (read-only).
- **Таблица лидеров** — рейтинг по звёздам (подиум + список).
- **Витрина клуба** — перки, открываемые по числу звёзд.
- **Бадди / Random Coffee** — подбор напарника раз в месяц (match-анимация).
- **Календарь** — расписание клуба, события, дни рождения; таймер «До встречи резидентов» по МСК.

### Админ-панель
- **Дашборд** — KPI (всего/активные/неактивные/вовлечённые), график запусков (14 дн.), тренды Retention/Churn (6 мес.), таблицы платежей (действующие/просроченные).
- **Участники (CRM)** — список + поиск, ручное добавление, карточка со всеми полями, загрузка фото, прогресс, срок доступа, достижения+роли, доступ к ресурсам, удаление.
- **Таблица лидеров** — подиум + список по звёздам, переход к карточке.
- **Геймификация** — редактирование каталога достижений (`Достижения`) и ролей (`Роли`).
- **Витрина клуба** — CRUD перков (доступ по звёздам).
- **Уведомления** — DM резидентам в бота (гибрид авто по расписанию + ручная отправка), картинки-баннеры, произвольные рассылки, приветствие в канал.
- **Тема** день/ночь, вход по паролю (`ADMIN_PASSWORD`).

### Бот `@apclubnews_bot`
- Приветствие на `/start` (баннер + текст + inline-кнопки «Вход в клуб» / «Меморандум»).
- DM-уведомления о событиях (эфиры, дни рождения, план/итоги недели) — авто по расписанию (cron) + ручные из админки.
- Приветствие в канал (url-кнопка, т.к. web_app в каналах запрещён).
- `getChat` для подтягивания `@username` резидентов (для прямых ссылок в админке).

## ⭐ Ключевые механики

### Звёзды и роли
`звёзды резидента = число полученных money-достижений + число ролей на Тире 5`.
У каждой роли 5 тиров; тиры 1–4 — только прогресс, звезду даёт Тир 5.
Единый расчёт — `computeStars(profile, catalog)` (продублирован в
`frontend/src/stars.ts` и `admin/src/stars.ts`, держать синхронно).
Обратная совместимость: legacy role-id в `achievements[]` = Тир 5.

### Уровни (статусы по месяцам)
Число уровней **динамическое** = длине списка названий (до 24). Прогресс растёт
автоматически по календарным месяцам с активации + ручная корректировка админа
(`setMonth`/`bonusMonths`). Названия статусов редактируются в карточке участника
(общие для всех), хранятся в служебной записи `__catalog.levels`.

### Доступ / подписка
`accessUntil` (timestamp) = дата следующего платежа; по истечении вход закрыт
(гейт во фронте — в планах). «Сделать бессрочным» очищает дату. Бейдж:
«активен до ДАТА» / «♾ бессрочно» / «истёк».

### Прямая ссылка в Telegram
Кнопка «Написать» у каждого резидента (список участников, карточка, лидеры,
дашборд). Приоритет ссылки: `@username` → `https://t.me/<username>` (надёжно) →
числовой id → `tg://user?id=` (ненадёжно, может открыть «Избранное»). Username
подтягивается автоматически: при входе в Mini App (`/api/launch`) и бэкофиллом
через бота (`getChat`) при загрузке списка в админке.

### Модель данных (Supabase `profiles`)
Одна таблица, профиль целиком в jsonb-колонке `data` (upsert = читаем→мержим→пишем).
Служебные записи — под ключами с префиксом `__` (фильтруются из списков):
`__catalog` (достижения+уровни), `__showcase` (перки витрины), `__stats`
(суточные/помесячные метрики), `__resources` (каталог ресурсов), `__notifications`
(конфиг рассылок). DDL (создание таблиц) — только через SQL Editor Supabase.

## 🚀 Запуск локально

Нужны **три процесса**. Node ≥ 22.6, npm 11.

```bash
# 1) API-сервер (:3000) — Supabase, бот, планировщик
cd server && npm install && npm run start

# 2) Mini App (:5173) — проксит /api на :3000
cd frontend && npm install && npm run dev

# 3) Админ-панель (:5174) — проксит /api на :3000
cd admin && npm install && npm run dev
```

Тестировать Mini App на **:5173** (не на preview :4173 — там нет прокси).
Прод-сборка: `npm run build` в каждой папке (→ `dist/`).

> ⚠️ Локальный сервер по умолчанию бьёт в **прод-Supabase** (креды в `server/.env`).
> Изменения через локальную админку меняют реальные данные. Для теста используйте
> временные записи и удаляйте их, либо поднимите отдельный проект Supabase.

## 🔐 Переменные окружения

**`server/.env`** (значения — в Vercel env проекта `apclub-api`, в репозиторий не коммитятся):

| Переменная | Назначение |
|------------|------------|
| `BOT_TOKEN` | Токен бота @apclubnews_bot |
| `SUPABASE_URL`, `SUPABASE_SERVICE_KEY` | Доступ к Supabase (нужен **секретный** `sb_secret_…`, не публичный) |
| `ADMIN_PASSWORD` | Пароль входа в админку |
| `ADMIN_IDS` | Telegram-id админов (через запятую) |
| `ALLOW_DEV_AUTH` | `1` — dev-вход без Telegram (**в проде `0`!**) |
| `CRON_SECRET` | Защита эндпоинта cron-уведомлений |
| `MINIAPP_URL` | URL Mini App (для кнопок бота в личке — web_app) |
| `MINIAPP_LINK` | Прямая ссылка Mini App (для кнопки в канале — url) |
| `CHANNEL_ID` | Канал клуба для приветствия (опц.) |

**`frontend/.env` и `admin/.env`:** `VITE_API_URL=https://apclub-api.vercel.app`

## 📦 Деплой

Все три проекта — на Vercel, из соответствующих папок:

```bash
cd server   && vercel deploy --prod --force   # apclub-api  (serverless Express)
cd frontend && vercel deploy --prod --force   # apclub      (статика)
cd admin    && vercel deploy --prod --force   # apclub-admin (статика)
```

- **`--force` обязателен** — Vercel кэширует сборку и обычный редеплой может отдать старую.
- Сервер под Vercel: `server/api/index.ts` (export app), `server/vercel.json`
  (rewrite `/api/*`, cron `/api/cron/notify`), бот — webhook (не polling).
- Хеши прод-бандлов **отличаются** от локальных (Vercel собирает у себя) — сверять по строкам кода, не по хешу.
- Env меняются через `vercel env add` (по одной), затем **форс-редеплой**.

Полный чек-лист — в [`DEPLOY.md`](./DEPLOY.md). Бэкенд-альтернатива (Render blueprint) — `render.yaml`.

- Воронка: регистрация → активность → удержание → оплата.
