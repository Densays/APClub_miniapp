# APClub · Админка (отдельное браузерное приложение)

Standalone-панель управления сообществом. Не связана с Telegram: обычный сайт,
который ты деплоишь отдельно (например, в личный кабинет apk-lab) и открываешь в
браузере. Управляет теми же участниками через тот же API-сервер и Supabase, что и
Mini App.

## Что умеет
- Вход по паролю (не Telegram).
- Список всех участников (включая скрытых), поиск по имени / @нику / id.
- На каждого участника:
  - установить прогресс разблокировки (0–12 месяцев);
  - выдавать/снимать достижения (22 шт., звёзды);
  - править имя/фамилию;
  - скрыть/показать профиль;
  - удалить участника (например, демо-резидентов).

## Авторизация
Панель логинится паролем `ADMIN_PASSWORD` (задаётся в `server/.env`). Сервер
возвращает токен, который хранится в `localStorage` и шлётся в заголовке
`x-admin-token`. Пароль по умолчанию в репозитории — **`apclub-admin-2026`,
обязательно смени его в `server/.env`.**

## Запуск локально
Нужен запущенный API-сервер (`../server`, порт 3000).

```bash
npm install
npm run dev        # http://localhost:5174 (проксирует /api → :3000)
```

## Сборка и деплой в apk-lab
```bash
cp .env.production.example .env.production   # вписать VITE_API_URL = домен API
npm run build                                # готовый сайт → dist/
```
Залей содержимое `dist/` как статический сайт в apk-lab (или любой хостинг
статики: Vercel/Netlify/Cloudflare Pages). Никакого сервера панели не нужно —
это чистая статика, которая ходит в твой API.

## Что нужно на стороне API (server) для прода
- `ADMIN_PASSWORD` — надёжный пароль панели.
- `ALLOW_DEV_AUTH=0` (или пусто) — **обязательно**. При `=1` включён dev-вход без
  проверки, и админ-роуты откроются без пароля. В проде выключи.
- API должен быть доступен по HTTPS с того домена, что указан в `VITE_API_URL`.
- CORS на сервере уже открыт для кросс-доменных запросов панели.
